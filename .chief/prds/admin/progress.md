## Codebase Patterns
- All Livewire pages use file-based components with `âš¡` prefix in `resources/views/pages/`
- Organization scoping pattern: `Auth::user()->currentOrganization()` then query by org's project IDs
- Sidebar is a Blade partial at `resources/views/layouts/app/sidebar.blade.php`, not a Livewire component
- Factories have `withOrganization()` state on UserFactory
- Feature tests use `$this->actingAs($user)` and route helpers
- Routes use `Route::livewire()` for full-page Livewire components
- Flux UI Free components used throughout (`flux:dropdown`, `flux:menu`, `flux:sidebar.item`, etc.)
- Run `vendor/bin/pint --dirty --format agent` after PHP changes
- All pages check `$this->organization` existence and show "No Organization" fallback
- Data test attributes used for testability (e.g., `data-test="org-switcher"`)
- Clickable items use `<a>` with `wire:navigate` and `hover:bg-zinc-50 dark:hover:bg-zinc-800/50` classes
- When creating routes for future stories, add placeholder pages so links work immediately
- Livewire tests use `Livewire::test('pages::component.name', [...])` â€” not ComponentRegistry
- Delete confirmation uses `flux:modal` with `flux:modal.trigger` and `flux:modal.close` pattern
- Livewire validation is usually inline; Agent CRUD uses Form Request classes (`StoreAgentRequest`, `UpdateAgentRequest`) with `getRules($orgId)` static method for Livewire context
- Tags stored as JSON array, edited as comma-separated string, converted with `explode/implode`
- Stories belong to Epics (not directly to Projects) â€” use `Epic::factory()->create(['project_id' => ...])` then `Story::factory()->create(['epic_id' => ...])`
- Bugs belong directly to Projects â€” `Bug::factory()->create(['project_id' => ...])`
- `withCount()` on Eloquent queries for efficient count columns in list pages
- Inline CRUD pattern: use `$editingId` state variable to toggle display/edit mode per row, `unset($this->computed)` to bust cache
- `.chief` directory is gitignored â€” don't try to `git add` PRD files
